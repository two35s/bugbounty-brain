# [www.drive2.ru] CSRF through FCTX token bypass

### SUMMARY BY DRIVE.NET, INC.

During login on the login page, login is attempted through the FCTX token. In addition, the login page was implemented through g-recaptcha-response captcha, but an attacker can bypass g-recaptcha-response captcha without FCTX tokens, and login CSRF is possible.

The issue was fixed by enabling the missing anti-CSRF token validation.

### RESEARCHER SUMMARY

Many developers do not implement CSRF tokens on their login page. This doesn't have a big impact on security, but the CSRF token is implemented in Drive2, but the CSRF token is missing and XSS linkage was possible during login. XSS is performed through the `rememberMe` variable during login.
