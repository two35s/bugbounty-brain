# SSRF (open) - via GET request

### SUMMARY BY VK.COM

Потенциальный DoS сервиса share через SSRF. 
Продемонстрирована возможность проведения атаки через приложения ВК.

### RESEARCHER SUMMARY

Сервис *upload.php* / *parse_share* игнорировал идентифицирующие ключи *hash* и *rhash*, а также не производил проверку *Content-** по целевому url. Большой таймаут серверного запроса позволял нагрузить себя значительным объемом информации (цикличные запросы) => DoS .
Опционально: Посредством другой уязвимости в модуле редактирования приложения */editapp*, запрос к выше упомянутому сервису можно было подсунуть в *https-iframe*. При заходе юзера в приложение он инициировал тяжелый запрос.
