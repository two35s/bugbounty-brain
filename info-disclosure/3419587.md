# AWS Auto Scaling Service Reporting "AWS Internal" for CloudTrail Events Generated from Specific Endpoints

### VULNERABILITY INFORMATION

When performing calls to the AWS API, [AWS CloudTrail](https://aws.amazon.com/cloudtrail/) is responsible for logging that API activity for customers. These CloudTrail event logs contain information about the API call, when it was performed, by whom, etc. 

**We have found 6 API endpoints for the AWS Auto Scaling service that incorrectly report the user-agent and network information as "AWS Internal".** As a result of this, an adversary can perform API calls using these endpoints and evade the logging of their IP address/operating system information. 

This is similar to previous bugs we've submitted for the following services:

- [Pinpoint SMS and Voice](https://hackerone.com/bugs?subject=user&report_id=3072841)
- [Kendra](https://hackerone.com/bugs?subject=user&report_id=3044471)
- [Comprehend Medical](https://hackerone.com/bugs?subject=user&report_id=2979238)
