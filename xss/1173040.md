_Steps:_

1-The affected program is enabled with a Hacker Facing Custom Fields feature:
https://hackerone.com/:handle/custom_fields

`Custom Fields`   >>   `Hacker Facing Custom Fields`  >>`new field with a title name (hello)`  >>    
`Select type : text`  >>  `Visibility; Hacker facing - this field will appear on the report for the hacker to complete`

2-Then a malicious user report a bug to that program and put this payload `"><img src=x onerror=alert(document.domain)>`
to Additional information field in the report and then submit the report.

3-when the program admin open the attacker report and doing this actions [edit or modify or change and save it] to the value of the attacker in report(Custom data - hacker facing)  field as a result a stored  xss issue happen.
